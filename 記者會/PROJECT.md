# 📋 PROJECT.md - 專案協作規範

> 這是「新聞記者會排程系統」專案的協作規範，AI 在修改此專案時應遵循以下規則。

---

## 🎯 專案概述

| 項目 | 說明 |
|------|------|
| **專案名稱** | 新聞記者會排程系統 (Press Conference Project) |
| **專案目標** | 讓所有工作人員透過網頁，即時查看自己在什麼時間點應該做什麼事 |
| **使用者** | 記者會現場工作人員（導播、攝影、主持人、場控等） |
| **技術棧** | HTML + CSS + JavaScript（純前端網頁） |

---

## ✅ 開發規範

### 程式碼撰寫
- [ ] **所有註解使用繁體中文**
- [ ] **變數和函式名稱使用有意義的英文命名**（如 `startTime`, `showSchedule`）
- [ ] **保持程式碼簡單**，優先使用基礎語法
- [ ] **避免使用複雜框架**，維持純 HTML/CSS/JS

### 檔案結構
- [ ] **主要入口檔案**：`index.html`
- [ ] **樣式檔案**：若有額外 CSS，放在專案根目錄或 `css/` 資料夾
- [ ] **腳本檔案**：若有額外 JS，放在專案根目錄或 `js/` 資料夾

---

## ✅ 修改規範

- [ ] **修改前不需要特別確認**（AI 有信心就可以直接做）
- [ ] **沒有禁止修改的區域**，整個專案都可以調整
- [ ] **修改後說明改了什麼、為什麼這樣改**

---

## ✅ 驗證規範

- [ ] **每次修改後，提供在瀏覽器中預覽的方式**
- [ ] **說明如何操作來測試新功能**
- [ ] **必要時提供截圖或錄影**來展示效果
- [ ] **確保在修改後網頁仍能正常顯示**

---

- [ ] 讓各角色（如導播、主持人）可以篩選查看自己的任務
